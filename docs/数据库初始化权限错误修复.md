# 数据库初始化权限错误修复

## 问题描述

执行 `python scripts/init_database.py` 时报错：

```
2026-03-24 17:32:39,791 - ERROR - 
❌ 初始化失败: 错误:  对模式 public 权限不够
LINE 2:         CREATE TABLE IF NOT EXISTS wells (
                                           ^
```

---

## 原因分析

### PostgreSQL 权限机制

在 PostgreSQL 中，权限授予分为两个层面：

1. **Template 数据库层面**（`postgres` 数据库）
   - 授予 `GRANT ALL ON SCHEMA public TO wellinfo`
   - 这仅在 `postgres` 数据库中生效

2. **目标数据库层面**（`wellinfo` 数据库）
   - 创建数据库时，会从 `template1` 复制权限
   - 但 `template1` 可能没有完整的权限设置
   - 需要在目标数据库中重新授予

### 问题根源

**`init_database.py` 只在 `postgres` 数据库中授予权限，但切换到 `wellinfo` 数据库后，权限没有正确继承！**

## 修复说明

### 关键改动

```python
# 授予 CREATE 权限（在目标数据库中重新授予）
cursor.execute(
    sql.SQL("GRANT ALL ON SCHEMA public TO {}").format(
        sql.Identifier(DB_USER)
    )
)
logger.info("✅ 授予模式权限 (在目标数据库中)")
```

**为什么需要这一步？**

1. **PostgreSQL 权限继承机制**
   - `CREATE DATABASE wellinfo` 会从 `template1` 复制
   - `template1` 的 `public` 模式可能没有授予 `wellinfo` 用户权限
   - 需要在目标数据库中显式授予

2. **权限范围**
   - `GRANT ALL ON SCHEMA public` 包括：
     - `CREATE` - 创建表、视图等
     - `USAGE` - 使用模式
     - 其他模式级权限

3. **最佳实践**
   - 在目标数据库连接后，显式确认权限
   - 避免"权限隐式传递"带来的不确定性

## PostgreSQL 权限知识

### 模式权限 (SCHEMA)

| 权限 | 说明 | 是否必需 |
|------|------|---------|
| `CREATE` | 在模式中创建对象 | ✅ 是 |
| `USAGE` | 访问模式中的对象 | ✅ 是 |
| `ALL` | 所有模式权限 | ✅ 推荐 |

### 表权限 (TABLE)

| 权限 | 说明 | 是否必需 |
|------|------|---------|
| `SELECT` | 查询数据 | ✅ 是 |
| `INSERT` | 插入数据 | ✅ 是 |
| `UPDATE` | 更新数据 | ✅ 是 |
| `DELETE` | 删除数据 | ✅ 是 |
| `TRUNCATE` | 清空表 | ⚠️ 可选 |
| `REFERENCES` | 创建外键 | ✅ 是 |
| `TRIGGER` | 创建触发器 | ✅ 是 |
| `ALL` | 所有表权限 | ✅ 推荐 |

### 序列权限 (SEQUENCE)

| 权限 | 说明 | 是否必需 |
|------|------|---------|
| `USAGE` | 使用序列（自增ID） | ✅ 是 |
| `SELECT` | 查询序列值 | ✅ 是 |
| `ALL` | 所有序列权限 | ✅ 推荐 |

## 最佳实践

### 1. 显式授权

❌ **不推荐**：
```sql
GRANT ALL ON SCHEMA public TO wellinfo;  -- 仅在 template1 中
CREATE DATABASE wellinfo;  -- 权限可能丢失
```

✅ **推荐**：
```sql
GRANT ALL ON SCHEMA public TO wellinfo;  -- 在 template1 中
CREATE DATABASE wellinfo;
GRANT ALL ON SCHEMA public TO wellinfo;  -- 在 wellinfo 中再次授予
```

### 2. 验证权限

```sql
-- 检查模式权限
SELECT nspname, rolname, privilege_type 
FROM pg_namespace 
JOIN pg_authid ON pg_authid.oid = ANY (nspacl::regrole[])
WHERE nspname = 'public';
```

### 3. 分阶段授权

```sql
-- 阶段1: 数据库级
GRANT CONNECT ON DATABASE wellinfo TO wellinfo;

-- 阶段2: 模式级
GRANT ALL ON SCHEMA public TO wellinfo;

-- 阶段3: 对象级（已创建的表）
GRANT ALL ON ALL TABLES IN SCHEMA public TO wellinfo;
GRANT ALL ON ALL SEQUENCES IN SCHEMA public TO wellinfo;

-- 阶段4: 未来对象（未来创建的表）
ALTER DEFAULT PRIVILEGES IN SCHEMA public 
GRANT ALL ON TABLES TO wellinfo;
ALTER DEFAULT PRIVILEGES IN SCHEMA public 
GRANT ALL ON SEQUENCES TO wellinfo;
```
