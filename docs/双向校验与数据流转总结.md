## 一、核心问题解答

### 1.1 数据落地策略

**问题**: 数据是先落地到数据库里再返回,还是不落地到数据库里呢?

**答案: 推荐使用混合模式(方案C)**

```
┌─────────────────────────────────────────────────────────────┐
│                   数据流转架构(混合模式)                       │
└─────────────────────────────────────────────────────────────┘

用户上传文件
    ↓
保存文件到存储 (storage/uploads/)
    ↓
AI处理 (OCR → 分类 → 抽取 → 校验)
    ↓
创建Document记录 (status='processing')
    ↓
插入ExtractedData记录 (批量插入,提高性能)
    ↓
插入QualityReport记录
    ↓
插入ValidationResult记录
    ↓
更新Document记录 (status='success')
    ↓
返回API响应 (包含处理结果) ← 数据已持久化
    ↓
(可选) 异步更新统计信息

关键点:
✅ 数据先落地到数据库,再返回
✅ 使用数据库事务保证一致性
✅ 批量插入提高性能
✅ 提供配置选项: save_to_db (默认true)
```

**配置选项**:

```python
# config/settings.py

class Settings(BaseSettings):
    # 数据存储配置
    SAVE_TO_DATABASE: bool = Field(True, description="是否保存到数据库")
    SAVE_DOCUMENT_METADATA: bool = Field(True, description="是否保存文档元数据")
    SAVE_EXTRACTED_DATA: bool = Field(True, description="是否保存提取数据")
    SAVE_QUALITY_REPORTS: bool = Field(True, description="是否保存质量报告")
```

**API调用示例**:

```python
# 默认:保存到数据库
POST /api/v1/extract/single
{
  "file": "...",
  "save_to_db": true  # 默认true
}

# 可选:不保存到数据库
POST /api/v1/extract/single
{
  "file": "...",
  "save_to_db": false  # 快速返回,不持久化
}
```

## 二、双向校验机制

### 2.1 校验层次结构

```
┌─────────────────────────────────────────────────────────────┐
│                    四层双向校验体系                          │
└─────────────────────────────────────────────────────────────┘

[1] 字段库 (config/field_schemas.py)
   ├─ 167个核心字段定义
   ├─ 8大类分类
   ├─ 字段类型、范围、必填性
   └─ 位置: 数据规范的源头

   ↓ 双向校验1: 字段定义 ↔ API Schema

[2] API Schema (schemas/models.py)
   ├─ 请求参数校验
   ├─ 响应数据校验
   ├─ Pydantic模型验证
   └─ 位置: API层数据规范

   ↓ 双向校验2: API Schema ↔ 数据库模型

[3] 数据库模型 (db/models.py)
   ├─ 9张核心表
   ├─ 表结构定义
   ├─ 外键、索引、约束
   └─ 位置: 数据库层规范

   ↓ 双向校验3: 数据库模型 ↔ 数据库表

[4] 数据库表 (PostgreSQL)
   ├─ 实际数据存储
   ├─ 数据完整性约束
   └─ 位置: 数据持久化
```

### 2.2 双向校验详情

#### 校验方向1: 字段库 → API Schema

**目的**: 确保请求参数符合字段库定义

```python
# schemas/models.py

class BaseProcessRequest(BaseModel):
    category: Optional[Literal["drilling", "mudlogging", ...]]

    @validator('category')
    def validate_category(cls, v):
        """校验分类是否在字段库中"""
        if v is not None and v not in FIELD_SCHEMAS.CATEGORY_MAP:
            raise ValueError(f"无效的分类: {v}")
        return v

    @validator('target_fields')
    def validate_target_fields(cls, v, values):
        """校验目标字段是否在字段库中"""
        if v is not None and 'category' in values:
            category = values['category']
            valid_fields = FIELD_SCHEMAS.get_field_names(category)
            invalid_fields = [f for f in v if f not in valid_fields]
            if invalid_fields:
                raise ValueError(f"无效的字段: {invalid_fields}")
        return v
```

**校验内容**:
- ✅ category 是否在 `FIELD_SCHEMAS.CATEGORY_MAP` 中
- ✅ doc_category 是否与 category 匹配
- ✅ target_fields 是否在指定分类的字段库中
- ✅ 字段值范围校验(基于字段库的 min_value/max_value)

#### 校验方向2: API Schema → 字段库

**目的**: 确保响应数据符合字段库定义

```python
# schemas/models.py

class ProcessingResponse(BaseResponse):
    extracted_fields: Dict[str, FieldExtractionResult]

    @validator('extracted_fields')
    def validate_extracted_fields(cls, v, values):
        """校验提取的字段是否符合字段库定义"""
        if 'document_info' in values and values['document_info']:
            category = values['document_info'].category
            if category:
                valid_fields = FIELD_SCHEMAS.get_field_names(category)
                invalid_fields = [f for f in v.keys() if f not in valid_fields]
                if invalid_fields:
                    raise ValueError(f"提取了无效字段: {invalid_fields}")
        return v
```

**校验内容**:
- ✅ extracted_fields 的字段名是否在字段库中
- ✅ confidence 是否在 0-1 范围内
- ✅ 统计数据一致性(total = valid + invalid)

#### 校验方向3: 字段库 → 数据库模型

**目的**: 确保数据库模型字段符合字段库定义

```python
# db/models.py

class ExtractedData(Base):
    field_name = Column(String(100), nullable=False)
    field_value = Column(Text)
    field_type = Column(String(20))
    confidence = Column(Numeric(5, 2))
```

**字段类型映射**:

```python
# config/field_schemas.py → db/models.py

TYPE_MAPPING = {
    "string" → String(500)       # 可变长字符串
    "float" → Numeric(10, 2)     # 高精度浮点数

}
```

**约束映射**:

```python

field_def.required → Column(nullable=False)
field_def.min_value → CHECK 约束
field_def.max_value → CHECK 约束
```

#### 校验方向4: 数据库模型 → 字段库

**目的**: 验证数据库中的数据完整性

```python
# db/crud.py

def validate_extracted_data_integrity(
    well_no: str,
    category: str
) -> Dict[str, any]:
    """
    验证提取数据的完整性

    检查:
    1. 必填字段是否完整
    2. 是否有无效字段(不在字段库中)
    """
    # 获取数据库中的字段
    all_extracted_fields = [...]

    # 获取字段库中的字段
    required_fields = FIELD_SCHEMAS.get_required_fields(category)
    all_fields = FIELD_SCHEMAS.get_field_names(category)

    # 检查缺失的必填字段
    missing_required = [
        field for field in required_fields
        if field not in all_extracted_fields
    ]

    # 检查无效字段
    invalid_fields = [
        field for field in set(all_extracted_fields)
        if field not in all_fields
    ]

    return {
        'missing_required': missing_required,
        'invalid_fields': invalid_fields
    }
```

## 三、数据库表结构

### 3.1 核心表设计

```
┌─────────────────────────────────────────────────────────────┐
│                    数据库表结构(9张表)                        │
└─────────────────────────────────────────────────────────────┘

1. wells (井信息表)
   ├─ id: 主键
   ├─ well_no: 井号(唯一索引)
   ├─ oilfield, block: 油田、区块
   ├─ well_type, well_pattern: 井别、井型
   ├─ latitude, longitude: 坐标
   ├─ drill_date, completion_date: 钻井日期、完井日期
   └─ 关系: 一对多 documents, extracted_data, quality_reports

2. documents (文档表)
   ├─ id: 主键
   ├─ well_no: 井号(外键 → wells)
   ├─ document_id: 文档ID(唯一)
   ├─ filename, file_path: 文件名、路径
   ├─ category: 一级分类(索引)
   ├─ doc_category: 二级分类
   └─ status: 状态(pending/processing/success/failed)

3. extracted_data (提取数据表)
   ├─ id: 主键
   ├─ document_id: 文档ID(外键 → documents)
   ├─ well_no: 井号(冗余字段,便于查询)
   ├─ field_name: 字段名(索引)
   ├─ field_value: 字段值
   ├─ field_type: 字段类型
   ├─ confidence: 置信度(索引)
   └─ 唯一约束: (document_id, field_name)

4. quality_reports (质量报告表)
   ├─ id: 主键
   ├─ document_id: 文档ID(外键 → documents)
   ├─ well_no: 井号(冗余字段)
   ├─ completeness, accuracy, consistency: 质量指标
   ├─ overall_score: 总分(索引)
   ├─ quality_level: 质量等级
   └─ issues, suggestions: 问题、建议(JSONB)

5. validation_results (验证结果表)
   ├─ id: 主键
   ├─ document_id: 文档ID(外键 → documents)
   ├─ field_name: 字段名
   ├─ passed: 是否通过
   ├─ error_type: 错误类型
   └─ error_message: 错误信息

6. review_records (审核记录表)
   ├─ id: 主键
   ├─ document_id: 文档ID(外键 → documents)
   ├─ reviewer_id: 审核者ID(外键 → users)
   ├─ original_value, corrected_value: 原值、修正值
   └─ comment: 审核意见

7. processing_logs (处理日志表)
   ├─ id: 主键
   ├─ document_id: 文档ID(外键 → documents)
   ├─ well_no: 井号(冗余字段)
   ├─ stage: 处理阶段(索引)
   ├─ status: 状态
   ├─ duration_ms: 处理时长
   └─ error_message: 错误信息

8. users (用户表)
   ├─ id: 主键
   ├─ username, email: 用户名、邮箱(唯一)
   ├─ role: 角色(admin/reviewer/user)
   └─ is_active: 是否活跃

9. statistics (统计数据表)
   ├─ id: 主键
   ├─ stat_date: 统计日期(唯一)
   ├─ total_documents: 总文档数
   ├─ avg_confidence: 平均置信度
   └─ manual_review_rate: 人工审核率
```

## 七、校验矩阵

### 7.1 完整的校验映射

```
┌─────────────────────────────────────────────────────────────┐
│                    校验映射表                               │
└─────────────────────────────────────────────────────────────┘

字段库字段 → API Schema → 数据库模型 → 数据库表
    ↓            ↓               ↓           ↓
WellNo      category         well_no      wells.well_no
            target_fields    field_name    extracted_data.field_name
            confidence       confidence    extracted_data.confidence
            validated        is_valid      extracted_data.is_valid

数据类型映射:
string     → str             → String(500)   → VARCHAR(500)
float      → float          → Numeric(10,2) → NUMERIC(10,2)
int        → int            → Integer      → INTEGER
date       → datetime        → Date         → DATE

约束映射:
required   → Field(...)      → nullable=False → NOT NULL
min_value  → ge=...          → CHECK(...)    → CHECK(...)
max_value  → le=...          → CHECK(...)    → CHECK(...)
```

