# AI 驱动油气井关键信息提取-落地方案

## 一、项目概述

油气井全生命周期产生**钻井、录井、测井、试油 / 测试、完井、地质、生产**等资料/文档，是勘探开发、储量评估、生产决策的核心资产。当前普遍存在**80% 以上为非结构化 / 半结构化数据**，分散在 PDF、Word、扫描件、Excel、图纸中，无法被系统直接读取与利用。

### 现状

- 文档**多源异构、标准不一**：不同区块、承包商、时期的文档格式混乱，井号、层位、参数命名不统一；
- 管理**分散孤岛**：文档散落在个人电脑、各业务系统，跨专业、跨部门调取困难，版本混乱；
- 处理**高度依赖人工**：分类、录入、校对、归档全靠人工，耗时、易错、成本高、追溯难；
- 系统**无法直连**：现有勘探开发、生产管理、数据中台系统无法直接读取非结构化文档，数据入库滞后、齐全率低、一致性差。



为提高油气行业资料管理效率，针对钻井、录井、测井、试油/测试、完井、地质、生产等油气井相关资料/文档，提出基于AI的非结构化资料关键数据结构化提取方案。方案旨在通过AI模型自动完成文档分类与信息提取，形成结构化数据，并支持一键导入业务系统。

### 技术背景

- 行业标准：严格遵循**SY/T 5788、SY/T 5089、SY/T 5132**等油气井数据标准，字段体系成熟可落地；
- 技术成熟：**OCR、文档分类、信息抽取、大模型语义理解**已在能源工程领域规模化验证，可实现私有化部署；
- 业务刚需：单井 / 多井批量资料快速入库、跨井对比、智能检索、合规归档成为生产与科研刚需。



## 二、核心应用场景描述

### 场景 1：单口井批量资料一键入库

- 流程：用户批量上传该井**全部资料**→AI 自动**分类**（七大类）→按标准字段**提取关键数据**→**校验清洗**→**人工审核**→一键写入业务系统 / 数据中台
- 价值：单井资料从 “人工整理 ” 变为 “AI 处理  + 审核”，实现**一次上传、全量利用**。

### 场景 2：多口井混合资料批量处理

- 流程：批量上传**多井混合资料**→AI 自动**识别井号**→按井**归组**→逐井执行分类→提取→校验→批量入库→生成多井汇总报告
- 价值：解决 “多井混传、人工分拣” 痛点。

### 场景 3：历史资料治理与补录

- 流程：批量导入历史纸质扫描件 / 电子老文档→AI 结构化提取→补齐缺失字段→修正冲突数据→标准化入库
- 价值：盘活存量资产，支撑老井潜力再评价、储量复算、方案优化。

## 四、核心处理流程

1. ‌**文档输入**‌
   - 支持批量上传各种格式的文档：PDF、Word、Excel、PPT、TXT等。
   - 建立文档预检机制，识别文档完整性与可读性。
2. ‌**AI驱动的核心处理**‌
   - ‌**文档类型识别与分类**‌：采用预训练模型与关键词匹配算法，将文档自动归类至预定义文档分类类之中。
   - ‌**井号识别与去重**‌：利用OCR和自然语言处理（NLP）技术，从文档名称或内容中提取`井号(Well Name)`。在多口井资料批量导入场景中，以井号作为文档聚合的唯一标识。
   - ‌**关键信息结构化提取**‌：针对每一类文档，依据行业标准字段库，训练特定NER（命名实体识别）模型，从非结构化文本中精准提取预定义的关键信息项。



## 五、落地路径

1. **需求与数据准备**（2 周）：梳理样本、明确字段、对接系统、制定规则；
2. **AI 模型构建**（4 周）：分类模型、抽取模型、专业词典、校验规则；
3. **系统开发与集成**（4 周）：上传入口、AI 服务、审核页面、入库接口、权限管理；
4. **测试与优化**（3 周）：单井 / 多井实测、准确率迭代、性能优化；
5. **试点上线**（2 周）：典型区块试点、培训、问题整改；
6. **全面推广**（持续）：全单位覆盖、历史资料治理、持续迭代。
