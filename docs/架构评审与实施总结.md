### 1.3 架构优化措施

#### 措施1: 简化前端设计

**原设计**: 完整的 Vue 3 + TypeScript 前端

**优化后**:
- MVP 阶段: 仅提供 API 接口 + 简单 Web 界面 (Streamlit)
- 后期扩展: 根据需求再开发完整前端

**收益**:
- 降低初期开发成本 60%
- 加快迭代速度
- 专注核心 AI 功能

#### 措施2: 简化向量数据库设计

**原设计**: 使用 Milvus/Chroma 向量数据库

**优化后**:
- MVP 阶段: 使用本地文件缓存 Embedding
- 后期扩展: 积累足够文档后引入向量库

**收益**:
- 降低系统复杂度
- 减少运维成本
- 加速上线

#### 措施3: 统一文档分类策略

**原设计**: Embedding + LLM 混合分类

**优化后**: 统一使用 LLM (Qwen2.5-14B) 进行零样本分类

**收益**:
- 代码更简洁
- 维护成本更低
- 准确率相当 (LLM 零样本效果已足够好)

#### 措施4: 完善数据库设计

**新增表**:
- 配置管理表 (可选,初期使用配置文件)
- 提示词模板表 (可选,初期使用 Python 文件)

**优化**: 保持简洁,初期使用配置文件,后期按需引入数据库配置管理

#### 措施5: 增强错误处理

**新增机制**:
- 重试机制 (LLM 调用失败自动重试)
- 降级策略 (Embedding 失败时使用关键词匹配)
- 详细错误日志 (使用 loguru)
- 自定义异常类

#### 措施6: 提示词配置化

**实现方式**:
- 提示词模板集中管理 (config/prompts.py)
- 支持动态加载
- 版本控制友好

## 四、关键设计决策说明

### 4.1 为什么简化前端?

**原因**:
1. 项目初期重点是验证 AI 模型效果
2. 前端开发成本高,迭代慢
3. API 测试工具 + 简单Web界面足以满足需求

**方案**:
- 开发阶段: 使用 Postman/Apifox 进行 API 测试
- MVP 阶段: 使用 Streamlit 快速搭建简单 Web 界面
- 正式阶段: 根据用户反馈再开发完整前端

### 4.2 为什么不使用向量数据库?

**原因**:
1. 初期文档分类使用 LLM 零样本,不需要向量检索
2. Embedding 缓存使用本地文件即可
3. 降低系统复杂度和运维成本

**未来扩展**:
- 积累足够文档后,再引入向量库做语义搜索
- 用于相似文档推荐
- 用于知识库检索增强 (RAG)

### 4.3 为什么使用单一 LLM 服务?

**原因**:
1. 通义千问 API 支持多种模型,统一调用接口
2. 降低代码复杂度
3. 易于切换不同模型 (通义千问 / DeepSeek / OpenAI)

**配置**:
- 分类: qwen2.5-14b-instruct (平衡速度与准确率)
- 抽取: qwen2.5-72b-instruct (高准确率)
- 井号: qwen2.5-7b-instruct (快速响应)

### 4.4 为什么使用 Docker 部署?

**原因**:
1. 环境隔离,避免依赖冲突
2. 快速部署,一键启动
3. 易于扩展,支持 Kubernetes

**优势**:
- 开发环境与生产环境一致
- 降低运维成本
- 便于 CI/CD 集成

## 五、风险评估与应对

### 5.1 技术风险

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|---------|
| OCR 识别准确率不足 | 中 | 中 | 使用 PaddleOCR 专业版,支持图像预处理 |
| LLM 抽取结果不稳定 | 中 | 高 | 设置低 temperature,使用确定性输出 |
| PaddleOCR 安装失败 | 低 | 中 | 提供详细的安装文档,使用 Docker 统一环境 |
| LLM API 调用失败 | 中 | 中 | 实现重试机制,提供降级策略 |

### 5.2 进度风险

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|---------|
| 开发进度延误 | 中 | 中 | 优先实现核心功能,非核心功能后置 |
| 人力不足 | 低 | 高 | 调整开发优先级,聚焦 MVP |

### 5.3 成本风险

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|---------|
| LLM API 成本过高 | 中 | 中 | 使用较小的模型进行分类,优化提示词减少 token |
| 存储成本过高 | 低 | 低 | 定期清理过期文件,使用对象存储 |
