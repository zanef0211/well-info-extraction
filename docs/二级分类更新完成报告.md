### 3. ✅ 大语言模型文档分类已更新

| 组件 | 更新内容 |
|------|---------|
| `models/document_classifier.py` | 完全重构支持二级分类 |

**核心变更:**

#### ClassificationResult 数据类
```python
@dataclass
class ClassificationResult:
    category: str              # 一级分类代码: drilling/mudlogging等
    doc_category: str         # 二级文档分类: 钻井设计/完井设计等
    confidence: float         # 置信度
    reason: str              # 分类依据
    suggested_fields: List[str]  # 建议提取的字段
```

#### DocumentClassifier 更新
- 使用新的 `FieldSchemas.CATEGORY_MAP` 和 `FieldSchemas.DOC_CATEGORIES`
- System Prompt 包含完整的二级分类列表（69个文档类型）
- 自动验证一级分类和二级分类的一致性
- 根据分类返回建议提取的字段列表

**支持的分类体系:**
- **一级分类（7个）**: drilling, mudlogging, wireline, testing, completion, geology, production
- **二级分类（69个）**:
  - 钻井资料: 10个类型
  - 录井资料: 10个类型
  - 测井资料: 9个类型
  - 试油-测试资料: 10个类型
  - 完井资料: 10个类型
  - 地质资料: 10个类型
  - 生产资料: 10个类型

## 新增的API功能

### 1. 文档分类接口
```python
POST /api/classify
Request: {
    "file_path": "path/to/document.pdf",
    "document_text": "文档内容...",
    "metadata": {"filename": "钻井设计.pdf"}
}
Response: {
    "success": true,
    "category": "drilling",
    "category_name": "钻井资料",
    "doc_category": "钻井设计",
    "confidence": 0.95,
    "reason": "文档标题和内容符合钻井设计特征...",
    "suggested_fields": ["WellNo", "RigModel", "BitProgram", ...]
}
```

### 2. 获取文档分类列表
```python
GET /api/categories
Response: {
    "success": true,
    "categories": {
        "drilling": ["钻井设计", "完井设计", "钻井日报", ...],
        "mudlogging": ["录井报告", "气测记录", "岩屑录井", ...],
        ...
    }
}
```

### 3. 处理文档（支持指定分类）
```python
POST /api/process
Request: {
    "file_path": "path/to/document.pdf",
    "category": "drilling",        # 可选：指定一级分类
    "doc_category": "钻井设计",     # 可选：指定二级分类
    "enhance_images": true,
    "clean_text": true
}
Response: {
    "success": true,
    "data": {...},
    "category": "drilling",
    "doc_category": "钻井设计",
    "classification_confidence": 0.95
}
```

## 问题排查

### 常见问题

1. **数据库字段不存在**
   - 运行迁移脚本: `python -m scripts.migrate_add_doc_category`
   - 或重新初始化数据库

2. **分类结果为空**
   - 检查 LLM 客户端是否正确初始化
   - 查看日志中的详细错误信息

3. **二级分类不匹配**
   - 确认一级分类和二级分类的映射关系正确
   - 检查 `FieldSchemas.DOC_CATEGORIES` 配置
