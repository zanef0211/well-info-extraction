# 文档分类体系更新说明

## 更新概述

根据《AI-驱动油气井关键信息提取.md》需求文档，对系统的文档分类体系进行了重构，实现了两级分类体系。

## 主要变更

### 1. 基本信息-字段库共享机制

**变更前：**
- 基本信息(basic)作为独立的第8个一级分类
- 其他7个分类与基本信息分离

**变更后：**
- 基本信息-字段库作为所有资料/文档类型的共享字段
- 任何文档都包含基本信息字段（23个）+ 分类特定字段

**基本信息字段（23个）：**
- WellNo（井号）
- DocCategory（文档分类）
- Oilfield（油田/区块）
- WellType（井别）
- WellPattern（井型）
- SpudDate（开钻日期）
- CompletionDate（完钻日期）
- WellCompletionDate（完井日期）
- DrillingDays（钻井周期）
- CompletionDays（完井周期）
- DesignDepth（设计井深）
- TotalDepth（完钻井深）
- BottomFormation（完钻层位）
- DrillingPurpose（钻探目的）
- Longitude（经度）
- Latitude（纬度）
- Operator（作业者）
- Contractor（承包商）
- DrillSite（井场位置）
- WaterDepth（水深）
- Platform（平台名称）
- CoordinateSystem（坐标系）
- Elevation（海拔）

### 2. 一级分类（7大类）

| 分类代码 | 分类名称 | 分类特定字段数 | 合并后总字段数 |
|---------|---------|--------------|--------------|
| drilling | 钻井资料 | 21 | 44 (23+21) |
| mudlogging | 录井资料 | 20 | 43 (23+20) |
| wireline | 测井资料 | 20 | 43 (23+20) |
| testing | 试油-测试资料 | 20 | 43 (23+20) |
| completion | 完井资料 | 21 | 44 (23+21) |
| geology | 地质资料 | 22 | 45 (23+22) |
| production | 生产资料 | 21 | 44 (23+21) |

### 3. 二级文档分类

每个一级分类下包含多个二级文档分类：

#### 钻井资料（10个）
- 钻井设计、完井设计、钻井日报、固井报告、
- 钻井工程总结、井身结构图、钻头程序表、
- 套管程序表、固井质量报告、复杂情况报告

#### 录井资料（10个）
- 录井报告、气测记录、岩屑录井、综合录井图、
- 钻时录井、荧光录井、钻井取心报告、
- 后效监测报告、槽面显示记录、气测组分分析

#### 测井资料（9个）
- 测井原始数据、测井解释报告、井壁取心报告、
- 声波测井、密度测井、中子测井、
- 电阻率测井、自然伽马测井、测井综合曲线图

#### 试油-测试资料（10个）
- 试油设计、试油报告、测试报告、DST测试、
- 常规试油、压裂试油、试油成果报告、
- 压力恢复测试、产能测试、流体性质分析报告

#### 完井资料（10个）
- 完井设计、完井总结、射孔数据、完井管柱图、
- 射孔报告、压裂施工设计、压裂施工总结、
- 完井工程质量报告、井下作业报告、投产报告

#### 地质资料（10个）
- 地质设计、地质总结、储量报告、地层分层表、
- 岩心描述、储层评价报告、构造图、
- 沉积相分析、砂体对比图、连井剖面图

#### 生产资料（10个）
- 生产日报、开发方案、措施报告、生产动态分析、
- 采油工艺设计、修井作业报告、增产措施报告、
- 试采报告、产能评价报告、开发调整方案

## 数据库变更

### Document 表新增字段

```sql
ALTER TABLE documents
ADD COLUMN doc_category VARCHAR(100);

COMMENT ON COLUMN documents.doc_category IS '二级文档分类: 钻井设计/完井设计/钻井日报等';

CREATE INDEX idx_doc_category ON documents(doc_category);
```

### 字段说明

- **category**: 一级分类 (drilling/mudlogging/wireline/testing/completion/geology/production)
- **doc_category**: 二级文档分类 (钻井设计/完井设计/钻井日报等)

## API 变更

### 字段Schema更新

#### config/field_schemas.py

新增方法和属性：

```python
class FieldSchemas:
    # 二级文档分类定义
    DOC_CATEGORIES: Dict[str, List[str]]

    # 获取所有字段（包含基本信息 + 分类特定字段）
    @classmethod
    def get_all_fields(cls, category: str) -> List[FieldDefinition]

    # 仅获取分类特定字段
    @classmethod
    def get_category_fields(cls, category: str) -> List[FieldDefinition]
```

### 提示词更新
