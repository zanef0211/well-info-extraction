## 一、项目背景与现状

### 1.1 行业背景

油气井全生命周期产生**钻井、录井、测井、试油/测试、完井、地质、生产**等大量资料/文档,这些资料是勘探开发、储量评估、生产决策的核心资产。

### 1.2 现状问题

| 问题类别 | 具体表现 | 影响 |
|---------|---------|------|
| **数据异构** | 80% 以上为非结构化/半结构化数据,分散在 PDF、Word、扫描件、Excel、图纸中 | 无法被系统直接读取与利用 |
| **标准不一** | 不同区块、承包商、时期的文档格式混乱,井号、层位、参数命名不统一 | 跨专业、跨部门调取困难 |
| **管理分散** | 文档散落在个人电脑、各业务系统,版本混乱 | 追溯难、共享难 |
| **依赖人工** | 分类、录入、校对、归档全靠人工 | 耗时、易错、成本高 |
| **系统隔离** | 现有勘探开发、生产管理、数据中台系统无法直接读取非结构化文档 | 数据入库滞后、齐全率低、一致性差 |

### 1.3 项目目标

通过AI技术自动完成文档分类与信息提取,形成结构化数据,实现"一次上传、全量利用",支持一键导入业务系统。

## 二、核心业务需求

### 2.1 三大应用场景

#### 场景 1: 单口井批量资料一键入库
**用户流程**:
1. 用户批量上传该井全部资料
2. AI 自动分类（七大类）
3. 按标准字段提取关键数据
4. 校验清洗
5. 人工审核
6. 一键写入业务系统/数据中台

**业务价值**: 单井资料从"人工整理"变为"AI处理+审核",提升效率。

#### 场景 2: 多口井混合资料批量处理
**用户流程**:
1. 批量上传多井混合资料
2. AI 自动识别井号
3. 按井归组
4. 逐井执行分类→提取→校验
5. 批量入库
6. 生成多井汇总报告

**业务价值**: 解决"多井混传、人工分拣"痛点。

#### 场景 3: 历史资料治理与补录
**用户流程**:
1. 批量导入历史纸质扫描件/电子老文档
2. AI 结构化提取
3. 补齐缺失字段
4. 修正冲突数据
5. 标准化入库

**业务价值**: 盘活存量资产,支撑老井潜力再评价、储量复算、方案优化。

## 三、功能需求

### 3.1 文档输入与预处理

| 功能项 | 需求描述 | 输入格式 | 输出 |
|--------|---------|---------|------|
| 文档上传 | 支持批量上传各种格式文档 | PDF、Word、Excel、PPT、TXT | 文档对象 |
| 文档预检 | 识别文档完整性与可读性 | 文档对象 | 可读性报告 |
| OCR识别 | 扫描件转换为文本 | 扫描件 PDF/JPG/PNG | 文本内容 |
| 版面解析 | 提取文本块和表格结构 | 文档对象 | 文本块、表格数据 |

### 3.2 AI 核心处理能力

#### 3.2.1 文档类型识别与分类
- **功能**: 自动判断文档属于8大类中的哪一类
- **输入**: 文档文本（前2000字）
- **输出**: 分类结果 + 置信度 + 判断依据
- **准确率要求**: 初期 ≥85%, 优化后 ≥90%

#### 3.2.2 井号识别与去重
- **功能**: 从文档中提取唯一 WellNo,支持多井混传场景
- **输入**: 文档全文
- **输出**: 井号列表 + 主要井号 + 置信度
- **覆盖格式**:
  - 中文名称: 塔中1井、苏里格6-3-5井
  - 字母数字: TS1-1、HZ21-3D、QHD32-6-1
  - 区块+序号: GS17-1-4、PL19-3-1
  - 带井字: YS108井、YQ10-1井

#### 3.2.3 关键信息结构化提取
- **功能**: 针对每类文档,按字段库精准提取各字段值
- **输入**: 文档全文 + 文档分类 + 表格数据
- **输出**: 结构化 JSON（字段值 + 置信度）
- **特点**:
  - 支持 null 值（文档中不存在的字段）
  - 保留原始单位,不换算
  - 数值不四舍五入
  - 支持表格数据解析（如生产记录表格）

### 3.3 校验与质量控制

| 校验类型 | 校验规则 | 输出 |
|---------|---------|------|
| **必填字段检查** | WellNo、DocCategory 必须非空 | 缺失字段列表 |
| **数值范围校验** | 产量、压力、深度等数值在合理范围内 | 超范围字段及建议 |
| **日期格式校验** | 支持 YYYY-MM-DD 等多种格式 | 格式错误字段 |
| **井号格式校验** | 长度2-30字符 | 格式异常提示 |

### 3.4 人工审核界面

- **功能**: 展示AI提取结果,支持人工修正
- **展示内容**:
  - 文档预览
  - 提取字段列表（高亮低置信度字段）
  - 校验错误提示
  - 原文位置关联
- **交互能力**:
  - 字段值编辑
  - 批量操作（接受全部/拒收全部）
  - 修正反馈收集（用于模型迭代）

### 3.5 数据导出与入库

| 功能 | 需求描述 | 输出格式 |
|------|---------|---------|
| 单文档导出 | 导出单个文档提取结果 | JSON/CSV |
| 批量导出 | 导出批量处理结果 | JSON/CSV/Excel |
| 按井分组导出 | 按井号归组后导出 | Excel（多sheet） |
| API接口 | 对接业务系统/数据中台 | REST API |

## 五、技术需求

### 5.1 技术栈选型

#### 5.1.1 整体技术路线

```
文档输入
   │
   ▼
【Stage 1】文档预处理 → OCR + 版面解析（非结构化→文本块）
   │
   ▼
【Stage 2】文档分类模型 → 判断文档属于8大类中的哪一类
   │
   ▼
【Stage 3】井号识别模型 → 从文档中提取唯一 WellNo
   │
   ▼
【Stage 4】关键字段抽取模型 → 按分类字段库精准提取各字段值
   │
   ▼
【Stage 5】校验与置信度评估 → 规则校验 + 模型置信度打分
   │
   ▼
结构化 JSON 输出（送人工审核 / 入库）
```

#### 5.1.2 模型选型

| 阶段 | 任务 | 推荐模型 | 理由 |
|------|------|----------|------|
| 文档预处理 | PDF/扫描件版面解析+OCR | **PaddleOCR v4** + **PyMuPDF** | 中文支持优秀,表格/图文混排识别准确率高 |
| 文档分类 | 8分类文本分类 | **BGE-M3** Embedding + 余弦相似度<br>或 **BERT-wwm-ext** fine-tune | BGE-M3 零样本效果好,可快速上线 |
| 井号识别 | NER 实体识别 | **正则规则 + ChatGLM3-6B / Qwen2.5-7B** | 井号格式相对规律,规则先行覆盖80%;LLM兜底 |
| 关键字段抽取 | 结构化信息抽取 | **Qwen2.5-72B / DeepSeek-V3** (API)<br>或私有化部署 **Qwen2.5-14B-Instruct** | 长文本理解能力强,支持复杂表格和专业术语 |
| 置信度评估 | 字段质量打分 | 规则引擎 + LLM 自评分 | 结合业务规则和模型输出 logprob |

#### 5.1.3 部署方式选择

```
数据敏感度高（甲方要求）？
    ├── 是 → 私有化部署 Qwen2.5-14B-Instruct（A100/H20 × 2）
    │         推理延迟 ~3s/文档,吞吐 ~20文档/分钟
    └── 否 → 调用 DeepSeek-V3 / Qwen-Max API
              成本 ~0.002元/千token,单文档 ~0.05元
```

### 5.3 提示词工程需求

#### 5.3.1 提示词设计原则

1. **角色定义**: 明确 AI 是"石油工程文档分析专家"
2. **任务约束**: 严格按字段库输出,不能臆造不存在的字段值
3. **格式强制**: 输出固定为 JSON,含 `value` 和 `confidence` 双字段
4. **缺省处理**: 文档中找不到的字段输出 `null`,不猜测
5. **专业词典注入**: 将行业术语、单位、层位名称注入 System Prompt

#### 5.3.2 需要设计的提示词

- **通用 System Prompt**: 定义角色、行为规则、专业词典
- **文档分类提示词**: 8大类分类任务
- **井号识别提示词**: 井号识别与归一化
- **7类文档字段抽取提示词**:
  - 钻井资料抽取提示词
  - 录井资料抽取提示词
  - 测井资料抽取提示词
  - 试油测试资料抽取提示词（重点关注产量数据精度）
  - 完井资料抽取提示词
  - 地质资料抽取提示词
  - 生产资料抽取提示词

## 十、风险与应对

| 风险类别 | 风险描述 | 影响 | 应对措施 |
|---------|---------|------|---------|
| **技术风险** | OCR 识别准确率不足（手写字体、模糊扫描件） | 低置信度字段多,人工审核量大 | 推荐使用 PaddleOCR 专业版,支持图像预处理 |
| **技术风险** | LLM 抽取结果不稳定（同一文档多次抽取结果不一致） | 数据不可信 | 设置低 temperature(0.05),使用确定性输出 |
| **数据风险** | 样本数据质量差（标注错误、样本覆盖不全） | 模型训练效果差 | 建立数据质量审核流程,定期清洗样本 |
| **业务风险** | 业务规则变化（字段定义、标准更新） | 系统失效 | 设计灵活的字段库管理模块,支持动态更新 |
| **安全风险** | 数据泄露（文档上传到云端 API） | 合规风险 | 提供私有化部署选项,支持离线部署 |
