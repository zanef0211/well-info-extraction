# 多格式支持实现总结

## 实现概览

根据README.md中的核心功能要求,现已完整实现对**PDF、Word、Excel、TXT**四种文档格式的支持。

## 新增文件

### 1. Word文档解析器 (`preprocess/docx_parser.py`)

**功能**:
- 提取Word文档中的所有文本内容
- 按段落提取文本
- 提取表格数据
- 获取文档元信息(标题、作者等)

**技术实现**:
- 使用 `python-docx` 库解析 `.docx` 文件
- 遍历段落和表格,合并文本内容
- 保留表格结构信息

**方法列表**:
- `extract_text()`: 提取完整文本
- `extract_paragraphs()`: 按段落提取
- `extract_tables()`: 提取表格数据
- `get_document_info()`: 获取文档信息

### 2. Excel文档解析器 (`preprocess/excel_parser.py`)

**功能**:
- 提取Excel文档中的所有文本
- 按工作表提取数据
- 提取单元格数据
- 获取Excel文件信息

**技术实现**:
- 使用 `pandas` 和 `openpyxl` 库解析Excel文件
- 支持多个工作表
- 将表格转换为可读的文本格式

**方法列表**:
- `extract_text()`: 提取完整文本
- `extract_sheets()`: 提取所有工作表
- `extract_sheet_by_name()`: 按名称提取工作表
- `extract_cells_with_data()`: 提取非空单元格
- `get_excel_info()`: 获取文件信息

### 3. 文本文件解析器 (`preprocess/txt_parser.py`)

**功能**:
- 提取纯文本文件内容
- 自动检测文本编码(支持UTF-8、GBK、GB2312等)
- 按行提取文本
- 提取段落
- 关键词搜索

**技术实现**:
- 支持多种中文编码格式
- 自动编码检测和回退机制
- 提供灵活的文本提取方法

**方法列表**:
- `extract_text()`: 提取文本(自动编码检测)
- `extract_lines()`: 按行提取
- `extract_paragraphs()`: 提取段落
- `get_file_info()`: 获取文件信息
- `search_text()`: 关键词搜索

## 文件格式支持矩阵

| 格式 | 扩展名 | 解析器 | OCR支持 | 表格提取 |
|------|--------|--------|---------|----------|
| PDF | `.pdf` | PDFParser | ✅ | ✅ |
| Word | `.docx`, `.doc` | DocxParser | ❌ | ✅ |
| Excel | `.xlsx`, `.xls` | ExcelParser | ❌ | ✅ |
| TXT | `.txt` | TxtParser | ❌ | ❌ |
| 图像 | `.jpg`, `.png`, `.bmp`, `.tiff` | OCREngine | ✅ | ❌ |

## 处理流程

### Word文档处理流程
```
上传 .docx 文件 → FileHandler验证 → DocxParser提取文本
→ TextCleaner清洗 → _process_text → 文档分类 → 信息提取 → 返回结果
```

### Excel文档处理流程
```
上传 .xlsx 文件 → FileHandler验证 → ExcelParser提取文本
→ TextCleaner清洗 → _process_text → 文档分类 → 信息提取 → 返回结果
```

### TXT文档处理流程
```
上传 .txt 文件 → FileHandler验证 → TxtParser提取文本(自动编码检测)
→ TextCleaner清洗 → _process_text → 文档分类 → 信息提取 → 返回结果
```

## 注意事项

1. **Word文档(.doc)**:
   - 仅支持 `.docx` 格式,旧的 `.doc` 格式需要额外转换
   - 表格内容会被转换为可读文本格式

2. **Excel文档**:
   - 公式不会被计算,只读取单元格值
   - 复杂的格式(合并单元格、图表)不会保留

3. **TXT文档**:
   - 自动编码检测优先使用UTF-8
   - 如果检测失败会尝试其他编码

4. **性能考虑**:
   - Excel大文件可能需要较多内存
   - 建议对超大文件进行流式处理(待实现)
